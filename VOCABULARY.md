# VOCABULARY

## Overview
Defines approved and restricted language across the product.

---

## Approved Terms

- UI (not interface unless formal)
- AI-powered
- Workflow
- Component
- System

---

## Avoid / Banned Terms

- “Cutting-edge”
- “Revolutionary”
- “Next-gen”
- “Seamless” (overused)

---

## Naming Conventions

### Components
- Use clear, functional names
  - Button / Primary Button
  - Card / Product Card

### Files
- kebab-case only
  - design-system.md
  - button-component.md

### Variables
- camelCase
  - primaryColor
  - spacingLarge

---

## Tone Consistency Rules

- Same term = same meaning everywhere
- No synonyms for core features

---

## Microcopy Rules

- Buttons: 1–2 words max
  - “Get Started”
  - “Save”

- Errors: clear + actionable
  - “Email is invalid”
