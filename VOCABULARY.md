# VOCABULARY

## Overview
Defines and enforces all language used across outputs.

This system ensures consistency, clarity, and precision in all communication.

---

## 1. APPROVED TERMS

- UI (use instead of "interface" unless formal context)  
- AI-powered  
- Workflow  
- Component  
- System  

---

## 2. BANNED TERMS

- Cutting-edge  
- Revolutionary  
- Next-gen  
- Seamless  

---

## 3. TERM MAPPING (CRITICAL)

Replace automatically:

- interface → UI  
- tool → system  
- feature → component  
- platform → system  
- solution → system  

---

## 4. USAGE RULES

- Use the same term for the same concept everywhere  
- Do not introduce synonyms for core terms  
- Avoid vague or abstract wording  
- Prefer clarity over creativity  

---

## 5. NAMING CONVENTIONS

### Components
- Use clear, functional names  
  - Button / Primary Button  
  - Card / Product Card  

---

### Files
- kebab-case only  
  - design-system.md  
  - button-component.md  

---

### Variables
- camelCase  
  - primaryColor  
  - spacingLarge  

---

## 6. MICROCOPY RULES

### Buttons
- Max 2 words  
- Action-oriented  

Examples:
- Get Started  
- Save  
- Continue  

---

### Errors
- Clear and actionable  

Examples:
- Email is invalid  
- Password is too short  

---

## 7. LANGUAGE STYLE RULES

- Use direct, simple language  
- Avoid filler words  
- Avoid exaggeration  
- Avoid marketing clichés  
- Be specific, not generic  

---

## 8. CONSISTENCY RULES

- Same word = same meaning  
- No variation across outputs  
- No rephrasing of core system terms  
- Terminology must remain stable across documents  

---

## 9. ENFORCEMENT RULES (NEW)

- Replace banned terms automatically  
- Reject outputs with undefined terminology  
- Apply term mapping before final output  
- Ensure all core terms match approved list  

---

## 10. UNKNOWN TERMS

If a term is not defined:

1. Replace with closest approved term  
2. Simplify wording  
3. Remove if unnecessary  

---

## 11. OUTPUT VALIDATION

Before publishing, verify:

- No banned terms used  
- All core terms are consistent  
- Language is clear and specific  
- No unnecessary variation in wording  

If validation fails:
→ Output must be revised  

---

## 12. FAILURE CONDITIONS

An output fails if:

- Banned terms are used  
- Terminology is inconsistent  
- Undefined terms are introduced  
- Language is vague or unclear  
- Multiple words are used for the same concept  

---

## FINAL PRINCIPLE

Language must be consistent to be scalable.

If terminology changes, meaning breaks.
